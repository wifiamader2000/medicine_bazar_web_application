# Import Full Bangladesh Medicine Dataset

This guide explains how to import the complete 30,000+ medicine database into the Medicine Bazar platform.

The current GitHub repository only contains the web application and import scripts. The full scraped dataset is hosted separately to keep the repository lightweight.

## 1. Where to download the dataset
The full dataset generated by the `bd-medicine-scraper` is hosted on Kaggle:
**Dataset Name**: Assorted Medicine Dataset of Bangladesh (or the official Kaggle link provided by the scraper author).
You need to log in to Kaggle and download the CSV files.

## 2. Required Files
After downloading, ensure you have the following CSV files:
- `medicine.csv` (Required)
- `generic.csv` (Highly Recommended: Provides categories, uses, and side effects)
- `manufacturer.csv` (Optional)
- `indication.csv` (Optional)
- `drug_class.csv` (Optional)
- `dosage_form.csv` (Optional)

## 3. Where to put the files
Copy the extracted CSV files into the following directory inside the Medicine Bazar project:
\`\`\`
scripts/fixtures/bd-medicine-sample/
\`\`\`
*(Overwrite the existing sample files).*

## 4. How to Import
Open your terminal, navigate to the root of the project (`medicine_bazar_web_application`), and run the dedicated import script:
\`\`\`bash
node scripts/import-bd-scraper.js
\`\`\`

This script will automatically:
- Map Kaggle CSV columns (e.g., `medicine type` -> `dosageForm`, `unit price` -> `mrp`) to the `Medicine Bazar` format.
- Merge therapeutic classes, indications, and side effects from `generic.csv` into the medicine data.
- Skip exact duplicates.
- Safely update `database/mb_products.json`.

## 5. Verification
After the script completes, it will output a JSON summary. Look for:
- `"importedCount"`: Should show the number of newly added medicines (e.g., ~30,000).
- `"currentProductCount"`: The new total number of products.

You can also run count checks by viewing the output or searching the website.

## 6. Troubleshooting

- **Encoding Issue / Broken Bangla Text**: Make sure the downloaded CSV files are saved with UTF-8 encoding. If Bangla text shows as `???`, open the CSV in Excel/Notepad, select "Save As", and choose UTF-8 format.
- **Missing Columns / Failed Rows**: Ensure the column headers in the CSV exactly match `brand name`, `medicine type`, `generic`, `strength`, `manufacturer`, `unit price`. If not, update the mappings in `backend/services/ProductImportService.js`.
- **Price Missing**: If a medicine does not have a price in the CSV, it will be skipped or imported with a 0 price (if active flag allows).
- **Generic Missing**: If `generic.csv` is not present, medicines will still import but without detailed clinical descriptions.
